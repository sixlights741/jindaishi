# 东北大学校史地图：维护与素材接入手册

这份手册用于回答三个实际问题：
1. 如何把单色背景换成“校园实拍虚化背景图”；
2. 其他小组完成手绘图后，你如何快速插入网站；
3. 如何避免常见素材路径和发布错误。
4. 如何使用管理员后台进行增删改。

---

## 一、目录结构速览

核心文件与目录：

- 页面样式：`public/styles.css`
- 地点数据：`data/places.json`
- 详情页模板：`src/views/place.ejs`
- 图片目录：`public/images/`
- 手绘图目录：`public/images/drawings/`

---

## 二、替换全站背景图（校园照片虚化版）

### 1) 准备素材

建议参数：
- 分辨率：不低于 `1920x1080`
- 单图大小：建议 `300KB ~ 1.5MB`
- 构图：校园全景/主楼/校门，避免过于杂乱

### 2) 放置文件

把你的背景图命名为：

`campus-bg.jpg`

放到：

`public/images/campus-bg.jpg`

### 3) 样式机制说明

系统已在 `public/styles.css` 中启用：
- 背景图层（自动铺满）
- 虚化与饱和度增强
- 暗色遮罩和光晕叠层

因此你只需“替换同名文件”，一般无需改代码。

### 4) 自定义虚化强度（可选）

在 `public/styles.css` 查找 `body::before`：
- `filter: blur(8px) saturate(1.2);`

可按需调节：
- 更虚化：`blur(10px)` 或 `blur(12px)`
- 更清晰：`blur(5px)`

## 三、插入其他组手绘图（推荐流程）

## 1) 收集并重命名

建议统一命名格式：

`地点id-序号.jpg`

例如：
- `lib-old-a.jpg`
- `lib-old-b.jpg`
- `main-building-01.png`

## 2) 放入手绘图目录

复制到：

`public/images/drawings/`

## 3) 修改地点数据

打开 `data/places.json`，找到目标地点，在该对象里编辑 `drawings` 字段。

示例：

```json
{
  "id": "lib-old",
  "name": "原图书馆旧址",
  "lat": 41.7662,
  "lng": 123.4289,
  "period": "1950s-1980s",
  "summary": "见证学校早期教学与科研资料积累的重要地点。",
  "content": "......",
  "image": "/images/drawings/lib-old-cover.jpg",
  "drawings": [
    {
      "url": "/images/drawings/lib-old-a.jpg",
      "caption": "原图书馆旧址手绘立面图"
    },
    {
      "url": "/images/drawings/lib-old-b.jpg",
      "caption": "原图书馆旧址平面草图"
    }
  ]
}
```

字段说明：
- `image`：详情页顶部主图
- `drawings`：下方画廊图（可多张）
- `drawings[].url`：图片访问路径（以 `/images/...` 开头）
- `drawings[].caption`：图片说明文字

## 4) 刷新页面验证

进入地图 -> 点击该地点 -> 检查：
- 主图是否显示
- 画廊是否出现
- 每张图说明是否正确

## 四、常见错误与排查

### 1) 图片不显示

排查顺序：
1. 文件是否确实在 `public/images/drawings/`
2. JSON 路径是否写成 `/images/drawings/xxx.jpg`
3. 扩展名是否一致（`.jpg` 和 `.png` 不可混）
4. 文件名是否大小写一致

### 2) 页面报错打不开

高频原因是 `data/places.json` 格式错误（如多逗号、少引号）。

建议：
- 修改 JSON 时使用 VS Code 自动格式化
- 每次只改一个地点，边改边刷新

### 3) 画廊不出现

`drawings` 为空数组时不会显示画廊，这是预期行为。

---

## 五、建议的协作规则（小组）

为避免混乱，建议你在组内约定：

1. 每个建筑给一个固定 `id`（不要改来改去）；
2. 所有图片先放共享网盘，最终统一由你拷到 `public/images/drawings/`；
3. 文案先在共享表格定稿，再录入 `data/places.json`；
4. 每次上线前备份 `data/places.json`。

- [ ] `public/images/campus-bg.jpg` 已替换为校园背景图
- [ ] 每个地点至少 1 张主图（`image`）

- [ ] `data/places.json` 能被正常解析

## 七、管理员后台使用说明（新增）

## 1) 设置管理员账号

编辑文件：`data/admin_ids.json`

示例：

```json
[
  "20230001",
  "20230009"
]
```

说明：
- 这些学号登录后将显示“管理员后台”入口；
- 若某账号已在 `users.json` 中存在，重新登录即可读取管理员权限。

## 2) 后台入口与功能

- 入口路径：`/admin`
- 主要功能：
  - 新增地点
  - 编辑地点（含主图、经纬度、文案）
  - 删除地点
  - 批量录入手绘图路径

## 3) 手绘图批量录入格式

在后台“手绘图列表”文本框按行填写：

`图片路径 | 图片说明`

例如：

```text
/images/drawings/main-building-01.jpg | 主楼南立面手绘图
/images/drawings/main-building-02.jpg | 主楼轴测草图
```

系统会自动转成 `drawings` 数组并在详情页画廊展示。

## 4) 推荐操作流程

1. 同组同学交付手绘图；
2. 你先把图片放到 `public/images/drawings/`；
3. 进入管理员后台编辑地点并粘贴手绘图列表；
4. 点击保存后立即在前台预览验证。
